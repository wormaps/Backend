## 1. 프로젝트 개요

WorMap은 사용자가 특정 장소를 선택하면, 해당 지역의 건물·도로·횡단보도·POI 같은 구조 데이터와 교통·날씨·혼잡도 같은 실시간 상태 
데이터를 결합해 3D Scene으로 재구성하고, 이를 탑뷰와 워크뷰로 직접 탐방할 수 있도록 만드는 디지털 트윈 프로젝트다.

기존 지도 서비스는 위치와 경로를 보여주는 데는 강하지만, 특정 장소의 구조적 특징이나 현장 분위기를 직관적으로 체험하기는 어렵다.

WorMap은 이러한 한계를 보완하기 위해 실제 장소의 구조와 상태를 3D 공간으로 재구성하고, 
사용자가 이를 직접 둘러보며 이해할 수 있도록 설계되었다.

핵심 목표는 단순히 위치를 보여주는 것이 아니라, 사용자가 특정 장소를 미리 체험하고 탐방할 수 있도록 만드는 것이다.

---

## 2. 한 줄 설명

현실 장소의 구조와 상태를 3D 공간으로 재구성해 직접 탐방할 수 있게 만드는 장소 탐방형 디지털 트윈 시스템.

---

## 3. 목표 사용자

- 특정 지역의 분위기와 동선을 파악하고 싶은 사용자
- 실제 장소를 3D 형태로 보고 싶은 사용자
- 디지털 트윈, GIS, 공간 시각화에 관심 있는 사용자

---

## 4. 핵심 가치

### 4-1. 장소를 미리 체험 가능

사용자는 실제로 방문하지 않아도 특정 장소의 분위기, 구조, 이동 동선을 미리 확인할 수 있다.

### 4-2. 지도보다 직관적

2D 지도보다 건물 배치, 도로 구조, 보행 흐름, 횡단보도, POI 등을 훨씬 직관적으로 이해할 수 있다.

### 4-3. 실시간 상태 반영

교통량, 날씨, 혼잡도 등 현재 상태를 반영하여 단순한 정적 지도보다 더 현실감 있는 경험을 제공한다.

### 4-4. 탑뷰 + 워크뷰 지원

사용자는 전체 지역 구조를 내려다보는 탑뷰와 실제 거리를 걷는 느낌의 워크뷰를 오가며 탐방할 수 있다.

---

## 5. 사용자 흐름

1. 사용자가 3D 지구본에서 가고 싶은 지역이나 장소를 선택한다.
2. 선택된 장소를 기준으로 주변 반경의 공간 데이터를 수집한다.
3. 건물, 도로, 횡단보도, POI 등 구조 데이터를 기반으로 정적인 3D Scene 자산(.glb)을 생성한다.
4. 생성된 Scene에 교통, 날씨, 장소 정보 같은 실시간 상태 데이터를 연결한다.
5. FE는 .glb + MetaData + Live API를 조합해 최종 장면을 렌더링한다.
6. 사용자는 탑뷰와 워크뷰를 통해 해당 지역을 직접 탐방할 수 있다.

## 6. 주요 기능

### 6-1. 장소 검색

- 장소명 검색
- 지역명 검색
- 유명 랜드마크 검색
- Google Places 기반 검색

예:

- 시부야 스크램블 교차로
- 도쿄타워
- 강남역

### 6-2. 3D Scene 생성

- 건물
- 도로
- 횡단보도

- POI
- 공원

- 거리 분위기 체험

### 6-5. 실시간 상태 반영

- 교통량
- 날씨
- 혼잡도
- 시간대
- 조명 상태
- 간판 및 색상 변화

## 10. Scene 구성 방식

### 10-1. 정적 Scene

정적 Scene은 장소의 공간 구조를 담당한다.

포함 요소:

- 건물
- 도로
- 보행로
- 횡단보도
- 공원
- POI

정적 Scene은 .glb 파일로 생성된다.

### 10-2. Meta Data

Meta Data는 objectId와 실제 데이터를 연결한다.

예:

- 어떤 도로가 어떤 교통 데이터를 사용하는지
- 어떤 건물이 어떤 건물 타입인지
- 어떤 POI가 어떤 장소 정보와 연결되는지

### 10-3. Live API

Live API는 교통, 날씨, 혼잡도, 장소 상태 같은 실시간 정보를 제공한다.

FE는 .glb + MetaData + Live API를 조합해 최종 장면을 렌더링한다.
